# TaskPilot — Автономный ассистент управления задачами из почты и голосовых заметок

**TaskPilot** — это агентная система, которая анализирует загруженные пользователем письма и голосовые сообщения, извлекает задачи, встречи, проблемы и решения, ведет актуальный бэклог с зависимостями и автоматически фиксирует текущие проблемы в задачах.

---

## Проблема

Руководители и специалисты тонут в почте: задачи теряются, зависимости забываются, а повторяющиеся проблемы не фиксируются. Нет системы, которая видела бы все входящие запросы, понимала контекст, помнила, что блокирует что, и извлекала уроки из прошлых ошибок.

**Глубинная проблема:** Люди вручную переносят задачи из писем в трекеры, вручную отслеживают зависимости и не накапливают знания о том, какие проблемы возникали и как их решали.

---

## Для кого

Руководители проектов, тимлиды и занятые специалисты, которые хотят делегировать агенту рутину по ведению бэклога и извлечению инсайтов из переписки.

---

## Ключевая особенность

Агент не просто распознает задачи, а **фиксирует текущую проблематику прямо в задаче**. Например, если в письме сказано "задача зависла, потому что нет ответа от смежников", агент добавит в задачу поле "проблема: ожидание ответа от смежников" и при следующем анализе будет проверять, не решилась ли проблема.

---

## Что делает PoC

### 1. Прием данных
Пользователь загружает файлы: письма (.txt/.eml) и голосовые (.mp3/.wav) — 5-10 тестовых файлов.

### 2. Транскрибация и анализ
- **Whisper** переводит аудио в текст
- **LLM извлекает:**
  - Задачи: что сделать, дедлайн, приоритет, исполнитель
  - Встречи: тема, дата, участники
  - Проблемы: "срыв из-за...", "зависло, потому что..."
  - Решения: "договорились впредь..."

### 3. Ведение бэклога (SQLite/JSON)
Каждая задача содержит:
- Название, дедлайн, приоритет
- Статус: новая / в работе / выполнена / заблокирована
- **Поле "проблема"** — заполняется агентом автоматически, если из сообщения видно, что задача сталкивается с трудностями
- Зависимости: задача А блокирует задачу Б

### 4. Автономное обновление
- Новое письмо о задаче → обновление статуса/дедлайна/проблемы
- Задача выполнена → разблокировка зависимых
- Изменение приоритета → переранжирование
- Если проблема решена (например, пришел ответ от смежников) → очистка поля "проблема"

### 5. Планирование встреч
- Проверка зависимостей перед созданием встречи
- Поиск слотов в тестовом Google Calendar
- Создание события только после подтверждения пользователем

### 6. База знаний (JSON)
- Хранение извлеченных проблем и решений
- При повторении проблемы — предложение применить сохраненное решение

### 7. Интерфейс (Streamlit)
- Загрузка файлов
- Просмотр текущего бэклога с полем "проблема"
- История действий агента
- Панель подтверждения создания встреч

---

## Что PoC НЕ делает

- **Реальная почта (IMAP)** — только загрузка файлов
- **Постоянный мониторинг 24/7** — только по запросу пользователя
- **Автоматические действия без подтверждения** — запись в календарь только после approve
- **Сложные графы зависимостей** — только линейные связи (А блокирует Б)
- **Мультиязычность** — только русский язык
- **Интеграция с реальными трекерами (Jira/Trello)** — бэклог хранится в JSON/SQLite
- **Обучение на поведении пользователя** — только явно извлеченные решения

---

## Стек технологий (предполагаемый)

- Python 3.10+
- Whisper (локальная транскрибация)
- SQLite / JSON для хранения
- Google Calendar API
- Streamlit для интерфейса
- LLM (через API или локально) для анализа текста
